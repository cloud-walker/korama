# Vision

## Forward-looking

Korama is built looking forward, not backward. It targets modern runtimes and the current direction of the ecosystem, and deliberately declines to support legacy environments. We would rather adopt where the platform is heading — modern ESM, the latest React major, current Node — than carry the weight of where it has been. When a decision trades broad backwards compatibility for a simpler, more modern foundation, that trade-off is intentional and aligned with this vision.

## Intentionally opinionated

Korama is intentionally opinionated. We prefer a small, curated surface over raw flexibility: where we build on a powerful tool, we wrap it behind our own API and hide its details rather than re-exposing everything it can do. Consumers get a focused, type-safe, well-themed subset that encodes our conventions, not the full configuration space of the underlying tool. When we constrain choice in exchange for consistency, ergonomics, and a coherent design, that trade-off is intentional and aligned with this vision.
